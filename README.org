#+AUTHOR: Michael Jones (YardQuit)

* Overview

Mule-Modal is an opinionated modal editing minor-mode for Emacs, that
leverages Emacs' own built-in functions wherever possible. There is no
overlay state — just a minor-mode keymap that toggles on and off.

** Design Philosophy

- Leverage Emacs native commands and built-in functions wherever possible
- Custom commands only where beneficial
- No overlay-based state tracking — just minor-mode keymaps
- Graceful handling of optional dependencies (smartparens, org-mode)
- Cross-platform support (Linux/Wayland/X11, macOS, Windows)
- Terminal mode (=emacs -nw=) support with DECSCUSR cursor sequences

* Installation

Place =mule-modal.el= at =~/.config/emacs/mule-modal/mule-modal.el=, then
add the following to your =init.el= or =config.el=:

#+begin_src emacs-lisp

  (let ((mule-path (expand-file-name "mule-modal/mule-modal.el"
                                     user-emacs-directory)))
    (if (file-exists-p mule-path)
        (load mule-path nil t)
      (message "WARNING: Mule modal module not found at %s." mule-path)))

  (mule-modal 1)

#+end_src

* Usage

- Press =C-g= to enter MULE Normal state.
- In Normal: =h=,=j=,=k=,=l= navigate; =i=,=I=,=a=,=A=,=o=,=O=,=c= enter Insert state.
- In Insert: Standard Emacs behavior, press =C-g= to return to Normal.
- State indicators show in modeline: =MULE[N]= = Normal, =MULE[I]= = Insert.

* Customizing Keybindings

You can override existing bindings or add new ones in your =config.el=
by modifying =mule-normal-mode-map=.

** Override an existing binding:

#+begin_src emacs-lisp

  ;; Replace the default behavior of ';' with consult-outline
  (when (boundp 'mule-normal-mode-map)
    (keymap-set mule-normal-mode-map ";" #'consult-outline))

#+end_src

** Unbind a suppressed key to fall through:

#+begin_src emacs-lisp

  ;; Make ',' fall through to the major mode instead of being ignored
  (when (boundp 'mule-normal-mode-map)
    (keymap-unset mule-normal-mode-map ","))

#+end_src

** Add a leader key map:

#+begin_src emacs-lisp

  ;; Leader Map Definition
  (when (boundp 'mule-normal-mode-map)
    (unless (boundp 'mule-leader-map)
      (defvar mule-leader-map (make-sparse-keymap)
        "Leader key map under SPC."))
    (keymap-set mule-normal-mode-map "SPC" 'mule-leader-map))

#+end_src

* Excluded Modes

MULE Normal state is permanently disabled in certain major modes where
key suppression would break core functionality. In these modes, MULE
stays in Insert state (passthrough), keeping MULE active but
non-interfering.

** Built-in Defaults

The following modes are excluded by default because they manage
subprocess interaction or terminal emulation where suppressing keys
via =suppress-keymap= would break functionality:

| Mode          | Reason                                                                 |
|---------------+------------------------------------------------------------------------|
| =comint-mode= | Process interaction — all input goes to subprocess. Covers derivatives |
|               | like =shell-mode=, =inferior-python-mode=, =ielm-mode=, =gud-mode=     |
|               | via =derived-mode-p=.                                                  |
| =term-mode=   | Full PTY terminal emulation — character-at-a-time, raw mode.           |
| =vterm-mode=  | Same as =term-mode= but faster implementation.                         |
| =eshell-mode= | Own command-line parser and input loop.                                |

Derived modes are caught automatically by the =derived-mode-p= check
in =mule--ensure-default-state=, so listing =comint-mode= alone covers
dozens of comint derivatives.

** Adding Additional Excluded Modes

Modes like =dired-mode=, =ibuffer-mode=, =magit-status-mode=, and
=epa-key-list-mode= are NOT excluded by default. In these modes, MULE
Normal state works (keys are suppressed, =hjkl= navigates), but some
users may prefer passthrough to retain native keybindings.

To add modes to the exclusion list in your =config.el=:

#+begin_src emacs-lisp

  (with-eval-after-load 'mule-modal
    (dolist (mode '(dired-mode ibuffer-mode magit-status-mode epa-key-list-mode))
      (add-to-list 'mule-excluded-modes mode)))

#+end_src

Or add a single mode:

#+begin_src emacs-lisp

  (with-eval-after-load 'mule-modal
    (add-to-list 'mule-excluded-modes 'pdf-view-mode))

#+end_src

Verify the current exclusion list:

#+begin_src emacs-lisp

  M-x eval-expression RET mule-excluded-modes RET

#+end_src

* Enter Key Rules (DWIM)

MULE's =mule-enter-dwim= command dispatches the Enter (RET) key based
on context. In editing modes, Enter is blocked to prevent accidental
line breaks. In non-editing modes, the native RET binding is invoked.

For =org-mode= and =markdown-mode=, MULE uses a rule-based dispatch
system to execute context-aware actions when Enter is pressed.

** Built-in Default Rules

The following rules are installed by default (controlled by
=mule-default-enter-rules-enabled=, which defaults to =t=):

#+begin_src emacs-lisp

  ;; Toggle checkboxes in org list items
  (mule-add-enter-rule item :checkbox org-toggle-checkbox)

  ;; Toggle TODO/DONE on org headlines
  (mule-add-enter-rule headline :todo-type mule-org-todo)

  ;; Follow links in org, markdown, or plain text
  (mule-add-enter-rule link nil org-open-at-point
                       markdown-follow-thing-at-point
                       browse-url-at-point)

#+end_src

Each rule specifies:
- =ELEMENT-TYPE= :: Org element type to match (e.g. =item=, =headline=, =link=)
- =PROPERTY= :: Org element property to check (e.g. =:checkbox=, =:todo-type=), or =nil= for no property check
- =COMMANDS= :: One or more commands tried in sequence (fallback chain). The first =fboundp= and =commandp= match wins.

** Rule Priority

Rules use =add-to-list= which prepends to the front of the list. The
first matching rule wins.

To prioritize a specific command for an element type, add it LAST in
your =config.el= so it appears at the front of the dispatch list:

#+begin_src emacs-lisp

  ;; These will be checked in this order:
  ;; 1. shr-follow-link-at-point (added last, checked first)
  ;; 2. org-open-at-point (added first, checked second)
  (mule-add-enter-rule link nil org-open-at-point)
  (mule-add-enter-rule link nil shr-follow-link-at-point)

#+end_src

To replace a default rule entirely, disable the built-ins first, then
add your own:

#+begin_src emacs-lisp

  (with-eval-after-load 'mule-modal
    (setq mule-default-enter-rules-enabled nil)
    (mule-add-enter-rule link nil shr-follow-link-at-point))

#+end_src

** Disabling Built-in Rules

To define your own rules entirely, disable the defaults:

#+begin_src emacs-lisp

  (with-eval-after-load 'mule-modal
    (setq mule-default-enter-rules-enabled nil))

#+end_src

** Adding Custom Enter Rules

Rules can be added from =config.el= using the =mule-add-enter-rule=
macro. Example:

#+begin_src emacs-lisp

  (with-eval-after-load 'mule-modal
    ;; Follow links with shr-follow-link-at-point as first choice
    (mule-add-enter-rule link nil shr-follow-link-at-point
                         org-open-at-point
                         browse-url-at-point)

    ;; Open agenda files from org-agenda table view
    (mule-add-enter-rule org-agenda-heading nil
                         org-agenda-do-default-action))

#+end_src

** Dispatch Order

=mule-enter-dwim= checks elements in this order:

1. =org-agenda-mode= RET binding (if in agenda)
2. Context element (most specific, e.g. a link inside a headline)
3. Parent element (e.g. the headline itself)
4. Ancestor elements (walking up the org element tree)
5. Non-editing mode native RET binding

All rules are checked at each level before moving to the next. This
ensures the most specific match fires first.

** Available Element Types

Common org element types for rules:

| Element Type | Useful Properties | Example Commands                           |
|--------------+-------------------+--------------------------------------------|
| =link=       | =nil=             | =org-open-at-point=, =browse-url-at-point= |
| =headline=   | =:todo-type=      | =org-todo=, =mule-org-todo=                |
| =item=       | =:checkbox=       | =org-toggle-checkbox=                      |
| =src-block=  | =nil=             | =org-edit-special=                         |
| =table-row=  | =nil=             | =org-table-next-row=                       |
| =plain-list= | =nil=             | =org-toggle-item=                          |

* Cursor Management

MULE manages cursor shapes independently for Normal and Insert states.
In terminal mode (=emacs -nw=), DECSCUSR escape sequences are used for
reliable cursor rendering across terminal emulators.

** Configuration

#+begin_src emacs-lisp

  (with-eval-after-load 'mule-modal
    (setq mule-cursor-normal 'box      ; box, bar, hollow, or nil for global default
          mule-cursor-insert '(bar . 2))) ; bar with width 2

#+end_src

** Terminal Compatibility

Some terminals do not support DECSCUSR or produce garbled output. MULE
maintains a denylist of terminal types where escape sequences are
suppressed:

#+begin_src emacs-lisp

  ;; View current denylist
  M-x eval-expression RET mule--decscusr-denied-terminals RET

  ;; Add a terminal to the denylist interactively
  M-x mule--add-denylist-entry RET

  ;; Remove a terminal from the denylist
  M-x mule--remove-denylist-entry RET

#+end_src

Default denied terminals: =dumb=, =linux= (framebuffer console).

* Clipboard Integration

MULE's =mule-yank= and =mule-yank-pop= commands integrate with the
system clipboard, falling back to the kill ring when no clipboard tools
are available.

** Detection

At load time, MULE checks for:
- =wl-copy= / =wl-paste= (Wayland)
- =xclip= / =xsel= (X11)
- =pbcopy= / =pbpaste= (macOS)
- Native GUI clipboard (graphical Emacs)
- Windows native clipboard

If no tools are found and Emacs is running in terminal mode, a warning
is shown once per session. Yank operations fall back to the kill ring
silently after the first warning.

* Input Method Management

MULE automatically saves and restores the active input method when
transitioning between Normal and Insert states. This is useful for
non-English keyboard layouts (e.g. Swedish Postfix).

- Entering Normal state: saves and deactivates the current input method
- Entering Insert state: restores the previously saved input method

This prevents input method key sequences from interfering with Normal
mode keybindings.

* Smartparens Integration

When =smartparens= is loaded, MULE automatically:
- Binds =C-g= in =smartparens-mode-map= and overlay keymaps
- Clears transient smartparens overlays on Insert-to-Normal transition
- Defers overlay cleanup via idle timer to avoid race conditions

No configuration needed — integration is automatic when both packages
are loaded. If smartparens is not installed, MULE functions normally
without errors.

* Position Tracking

=M-x mule-jump-back= (bound to =S=) rotates through a ring of
previously recorded cursor positions. Positions are tracked
automatically via =post-command-hook= whenever the cursor moves to a
different buffer or position.

- Ring size controlled by =mule-position-ring-max= (default: 10)
- Skips markers whose buffer has been killed
- Independent of the mark ring and region

* Org-Scratch Buffer

=M-x mule-org-scratch= creates or switches to a scratch buffer in
=org-mode=. Useful for quick note-taking without creating a file.
